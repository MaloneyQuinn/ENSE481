◆ cli_receive()

int cli_receive ( char BUFFER[],
int CLI_COUNTER )

Logic for user input.

uses usart_get() to get user input and compares what was entered. Special cases are: Enter-> submits the BUFFER[] object and determines what was entered, Backspace-> removes last input from BUFFER, [ and ]-> [ decreases pwm duty cycle and ] increaes it.

Parameters
BUFFER(char array) keeps track of what the user has currently entered in the terminal.
CLI_COUNTER(int) keeps track of how many characters are in the BUFFER. if there are 0 nothing should be erased on a backspace.
Returns
cli_return (enum) defined in constants.h an enum is used for the variable options that can be returned.

◆ cli_update()

void cli_update ( uint32_t duty_cycle,
int adc_reading )

Updates the data section of the CLI interface.

Uses usart_print() to move the cursor and print updated values for the duty cyle of the servo as well as the analog reading of the IR sensor. It then restores the cursor to where the user last had it.

Parameters
duty_cycle(int) the value of the duty cycle from Timer4 Ch2 used for the servo. Measured in micro-seconds.
adc_reading(int) the value of the adc1 data register attached to the IR sensor. Takes the voltage and converts to a binary number.

◆ USART2_IRQHandler()

void USART2_IRQHandler ( void )

USART2 interrupt handler. Triggers on input and processes the input.

Takes value and inserts it into buffer and processes input. Lower priority than SysTick handler so it cannot act before Systick and will be pre-emted by SysTick handler.
